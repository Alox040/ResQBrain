# Target Users

## Primary User Groups
- **Organization Administrator**: configures Organization scope, governance settings, and Release policies.
- **Medical Content Author**: creates and updates Algorithms, Medications, Protocols, and Guidelines.
- **Medical Reviewer/Approver**: validates clinical correctness and approves content for Release.
- **Operations Lead**: aligns content with Region and County operating realities.
- **EMS Provider**: consumes approved content in operational use.

## Secondary User Groups
- **Regional Program Manager**: tracks cross-Organization differences and rollout readiness.
- **Quality and Compliance Stakeholder**: audits Version history, ApprovalStatus, and change rationale.
- **Product Strategy Team**: uses SurveyInsight data for roadmap prioritization.

## Access Model Expectations
- Users are assigned one or more UserRoles per Organization.
- Permissions are policy-driven and scoped to Organization context.
- Content visibility follows Role and Permission rules plus release state.

## Design Implication
The platform must optimize for safe authoring and review workflows as much as it does for frontline content consumption.
