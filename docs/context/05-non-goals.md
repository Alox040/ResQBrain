# Non-Goals

## Explicit Non-Goals
- Defining a specific UI navigation, screen hierarchy, or client layout.
- Committing to a specific framework, runtime, or backend stack.
- Preserving prototype-era folder assumptions as architectural rules.
- Treating mock or static demo content as production content governance.
- Embedding hardcoded Algorithms, Medications, Protocols, or Guidelines in application logic.
- Designing a single-tenant platform.

## Legacy Assumptions Removed
- UI-first architecture ownership.
- Platform coupling to Expo or mobile-first constraints.
- Duplicate app root assumptions and temporary prototype structures.
- Implicit trust in undocumented or non-versioned content changes.
